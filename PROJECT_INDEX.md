## Summary

Exile Assistant (repo: POEassist) is a single-page, static GitHub Pages app for Path of Exile 2: gear resistance checking (with OCR), a build shopping list, an atlas farm planner with a market radar, a Divine Market price-analytics tab, and an Exile Hub status dashboard. No backend — data is either in the browser's `localStorage` or in static JSON files refreshed by GitHub Actions.

## Tech style

- One big file: `index.html` (~5600 lines as of 2026-07-12) — inline `<style>`, inline `<script>`, no build step, no framework.
- Static JSON in `data/` is the only "backend": generated by Node scripts in `scripts/`, committed by a GitHub Actions workflow.
- User-entered data (gear presets, shopping list, farm cards, filters) lives only in browser `localStorage` — never sent anywhere.
- Thai-language comments/strings are intentional throughout `index.html` — keep that style when editing nearby code.

### Shopping List (`shop`, aka Build Shopping List)
- Markup: `<div id="pageShop">` (~2170; summary cards + buy-next strip + `.shop-layout` two-column: left `#shopGearPanel` gear board / right toolbar+sort+quick-filter chips+table)
- Nav button: `tabBtnShop`
- JS section: `==================== Build Shopping List ====================` (~3360)
  - `ITEM_TYPES`, `SHOP_SLOTS` (equipment-slot list — slot ids are persisted in `row.slot`, never rename existing ids), `SHOP_ROWS_KEY`/`SHOP_RATE_KEY`
  - `newShopRow()` (row shape incl. additive `name` 2026-07-12 and `slot` 2026-07-12 fields), `rowSlotId()`/`rowMatchesSlot()`, `updateShopSummary()`, `renderShopBuyNext()`, `rowEstPrice()`, `sortShopRowsForDisplay()`, `buildLinkCell()` (link-chip rendering), `renderShopTable()`
  - Gear board: `buildShopGearPanel()` (built once at init), `updateShopGearPanel()` (per-slot count/bought/remaining/missing in the sub line + coverage state classes `cov-empty`/`cov-planned`/`cov-partial`/`cov-complete`/`cov-warn` — refreshed from `updateShopSummary()`), `selectShopSlot()`; slot buttons are `div[role=button]` (so a corner `+` quick-add `<button>` can nest) built by `makeShopSlotBtn(id, label, fullLabel, wide, iconSvg)` — uses **readable full labels** (additive `board` label on long slots, ids unchanged) + original per-slot inline icons from `SHOP_SLOT_ICONS`. `shopSelectedSlot` state is `all | unassigned | <slot id>`, in-memory only
  - Quick-add: `quickAddToSlot(slotId)` (adds a row into a slot from the board `+` / empty-state button, switches view, focuses name via `focusShopRowName()`, resets filters via `resetShopViewFilters()`)
  - Table sizing (Excel-like, patch 0.11): `SHOP_COLS`/`SHOP_ROW_H` defs, `applyShopSizing()`, `startShopColResize()`/`initShopColResize()` (drag `<th> .col-resize` handles), `stepShopRowHeight()`, `resetShopSizing()`; table is `table-layout:fixed` + a `<colgroup>` with stable `data-col` ids, row height via `--shop-row-h`.   - Numpad: `shopNumpadEl`/`openShopNumpad()`/`shopNumpadPress()` — on-screen numpad for every `input.num-input` in `#pageShop`; those inputs are `type=text inputmode=decimal` (NOT `type=number` — number inputs reject intermediate values like `3.`)
- Data source: none; rows in `localStorage` (`SHOP_ROWS_KEY`).
- Safe edit notes: link fields must stay chip-style (`buildLinkCell`), not raw URLs. Remaining ("ต้องใช้อีก") sums unbought rows only; per-slot remaining on the board uses the same min??max rule. Quick filter/sort/slot view are display-only — never reorder `shopRows`; handlers use `shopRows.indexOf(row)`. Old rows without `slot` load as Unassigned (tolerant `Object.assign` read) — storage key unchanged. Buy-next is global across all slots by design. `.shop-layout`'s mobile breakpoint must keep `minmax(0,1fr)` (plain `1fr` overflows via the table's min-width). See `EDIT_GUIDE.md` Feature Map row for full notes.
