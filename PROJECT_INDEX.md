# PROJECT_INDEX.md — POEassist / Exile Assistant

Read this before opening any large file. Do not read everything — jump straight to the line ranges below.

## Summary

Exile Assistant (repo: POEassist) is a single-page, static GitHub Pages app for Path of Exile 2: gear resistance checking (with OCR), a build shopping list, an atlas farm planner with a market radar, and an Exile Hub status dashboard. No backend — data is either in the browser's `localStorage` or in static JSON files refreshed by GitHub Actions.

## Tech style

- One big file: `index.html` (~4700 lines) — inline `<style>`, inline `<script>`, no build step, no framework.
- Static JSON in `data/` is the only "backend": generated by Node scripts in `scripts/`, committed by a GitHub Actions workflow.
- User-entered data (gear presets, shopping list, farm cards, filters) lives only in browser `localStorage` — never sent anywhere.
- Thai-language comments/strings are intentional throughout `index.html` — keep that style when editing nearby code.

## File map

| File | Role |
|---|---|
| `index.html` | Entire frontend: markup, CSS, JS for all 5 tabs |
| `README.md` | Placeholder (just a title) — not a source of project info |

| `data/market-radar.json` | Generated poe.ninja snapshot for Market Radar |
| `data/icon-map.json` | Currency/menu icon lookup, with emoji fallback |
| `scripts/update-home-status.mjs` | Generates `data/home-status.json` (scrapes poe2db.tw) |
| `scripts/update-market-radar.mjs` | Generates `data/market-radar.json` (fetches poe.ninja) |
| `.github/workflows/update-market-radar.yml` | Runs both scripts hourly + on push to `main`, commits data if changed |
