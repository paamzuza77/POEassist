## Summary

Exile Assistant (repo: POEassist) is a single-page, static GitHub Pages app for Path of Exile 2: gear resistance checking (with OCR), a build shopping list, an atlas farm planner with a market radar, a Divine Market price-analytics tab, and an Exile Hub status dashboard. No backend — data is either in the browser's `localStorage` or in static JSON files refreshed by GitHub Actions.

## Tech style

- One big file: `index.html` (~11290 lines as of 2026-07-14, patch 0.28) — inline `<style>`, inline `<script>`, no build step, no framework.
- Static JSON in `data/` is the only "backend": generated by Node scripts in `scripts/`, committed by a GitHub Actions workflow.
- User-entered data (gear presets, shopping list, farm cards, filters) lives only in browser `localStorage` — never sent anywhere.
- Thai-language comments/strings are intentional throughout `index.html` — keep that style when editing nearby code.

- **3 · Micro-interactions** — `uxFlash(el)` (green fade on a newly added/copied row/card; animates `background-color` only so tables don't jitter; `tr` is handled by animating its `<td>`s) and `uxBusy(btn, on, busyLabel)` (disable + `.is-busy` while Export/Import runs, restores the original label). Plus an app-wide `:focus-visible` ring and a pressed state on `.ocr-btn`. **Everything is behind a `prefers-reduced-motion` guard.**
- **Where they're used:** toasts on copy (Settings email + Keyword Codex), Export/Import (Settings backup, Gear Planner, Farm Planner), row/card add·copy·delete, Clear All, Market Radar refresh, OCR failure, prefs saved (`poeassist:prefschange`), Keyword-compare-full, Divine add-price. Empty/loading/error states on Gear Planner (no rows / empty slot / no filter match), Farm Planner (no cards / no match), Keyword Codex (no favorites **and** no results are distinct states), Content Codex, Market Radar. Flash on Gear Planner rows (via `focusShopRowName` — the shared add path), Farm cards (`flashFarmListItem`, needs the `data-farm-id` on `.farm-list-item`), Run Tracker rows, Resistance Checker stat rows.

The Danger Zone deliberately has **no** toast — it reloads immediately, so one would never be seen.
