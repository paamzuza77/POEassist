# EDIT_GUIDE.md

## Purpose

Tells future Claude Code sessions exactly where each POEassist / Exile Assistant feature lives, so they can patch the right lines in `index.html` (or the right data/script file) without reading the whole repo.

## Golden Rules

- Read `PROJECT_INDEX.md` first.
- Use this `EDIT_GUIDE.md` before opening source files.
- Use `rg`/grep before opening files — never scan `index.html` top to bottom.
- Open only the relevant line range (`Read` with `offset`/`limit`).
- Make minimal patches — no drive-by refactors.
- Update `PROJECT_INDEX.md`/`EDIT_GUIDE.md` when a feature is added or its anchors move.
- Verify with the smallest reasonable check (see each recipe below).
- Commit and push only after verification passes.

## Project Structure Map

This is a static, single-page, no-build-step app — there is no `src/`, `components/`, or `pages/` split. **Almost** everything UI-related lives inside one file; patch 0.34 began pulling self-contained helpers out into `js/`+`css/` (see "Modularization / helper files" below), but there is still **no bundler and no ES modules**.

- `index.html` — the app: all markup (one `<body>`), most CSS (one `<style>` block), and the main JS (one `<script>` block at the bottom). Two helper systems were extracted to `js/`+`css/` in patch 0.34 (below); everything else is still inline here.
- `js/ux-foundation.js`, `js/asset-registry.js` — extracted helper systems (patch 0.34), plain classic scripts `<script src>`'d **before** the main inline script. See "Modularization / helper files".
- `css/ux-foundation.css`, `css/asset-icons.css` — the matching styles, `<link>`ed after the inline `</style>`.
- `css/modern-theme.css` — the **"Modern" theme** (patch 0.35, the default): token blocks + all structural overrides, every rule scoped `[data-theme="modern"]`; `<link>`ed **last** so it wins the cascade. See "Styling / Theme Notes".
- `data/` — static JSON "backend": `home-status.json`, `market-radar.json`, `icon-map.json`.
- `scripts/` — Node generators that write the files in `data/`: `update-home-status.mjs`, `update-market-radar.mjs`.
- `.github/workflows/update-market-radar.yml` — runs both scripts hourly + on push, commits `data/*.json` if changed.
- No `package.json` / framework — plain HTML/CSS/JS, no npm install needed to run it (just open/serve `index.html`).
- Root docs: `PROJECT_INDEX.md` (repo map), `CLAUDE.md` (session rules), `EDIT_GUIDE.md` (this file).

## Modularization / helper files (patch 0.34, Phase 1)

Gradual modularization has started. **This is still a static GitHub Pages app with no build step — do not add npm/Vite/bundlers or convert to ES modules.** Two self-contained helper systems were pulled out of `index.html` verbatim:

| Helper system | Now lives in | Loaded / linked | API (unchanged) |
|---|---|---|---|
| **Modern UX Foundation** (patch 0.31) | `js/ux-foundation.js` + `css/ux-foundation.css` | JS as classic `<script src>` **before** the main inline `<script>`; CSS `<link>`ed right after the inline `</style>` | `showToast`/`showSuccessToast`/`showWarningToast`/`showErrorToast`/`showInfoToast`, `uxEmptyState`, `uxSkeleton`, `uxFlash`, `uxBusy` (+ `toastStackEl`/`dismissToast`) |
| **Game asset registry** (patch 0.32) | `js/asset-registry.js` + `css/asset-icons.css` | same (loaded before main script) | `renderAssetIcon`, `getLocalAssetForName`, `normalizeAssetKey`, tables `GAME_ASSETS`/`ASSET_ALIASES`/`RADAR_ASSET_BY_KEY`, broken-image `error` listener |

**How the scope works (important):** the two JS files are **plain classic scripts, not modules**. Multiple classic scripts on a page share one global lexical environment, so the top-level `const`/`function` declarations in these files are visible to the main inline script **as long as they load first** — which they do (their `<script src>` tags sit above the main `<script>`). So call sites in `index.html` use `showToast(...)`/`renderAssetIcon(...)` exactly as before, no `window.` prefix, no import. **Keep the load order** (helper `<script src>` before the main `<script>`); if you add another helper file, put its tag in the same spot.

- To **edit** toast/state/asset behavior, edit the external file — not `index.html` (which only keeps a pointer comment where each block used to be).
- To **add a new global helper** in this style: create `js/<name>.js` (plain top-level `const`/`function`s), add its `<script src>` before the main `<script>`, and optionally `css/<name>.css` `<link>`ed after `</style>`. Don't wrap it in a module or IIFE if the main script needs to call it.
- **Deferred to Phase 2 (do not force):** the tooltip systems stayed in `index.html` — `initAcHelpTips` (Augment Calc `?` tips) depends on `uiZoom()`/forge totals, and the **Keyword Help Bridge** tooltip is bound to the bridge block that **must stay first in the script** (its `kwHelpData()` TDZ guard around the end-of-file `let kwcData`). Extracting these safely needs a bigger plan. See `TODO.md`.
- **Beware `</style>` / `</script>` inside comments:** an HTML parser closes a `<style>`/`<script>` raw-text element on the literal end tag **even inside a CSS/JS comment**. When leaving a pointer comment inside the inline `<style>` or `<script>`, never write the literal `</style>`/`</script>` in it (write "style block" / "script src" in prose instead).

## Planner-first page reflow pattern (patch 0.42 — Gear Planner was round 1)

Each page should lead with the user's **next decision/action**, not with all data at once. The Gear Planner reflow (patch 0.42) is the reference implementation; apply the same structure one page per pass (queued in `TODO.md` → "Planner-first page reflow — next pages": Market Radar, Farm Planner, Resistance Checker, Content Codex, Keyword Codex).

The four surfaces, top to bottom:

1. **Hero / command area** — answers the page's first question in one glance (Gear: "ซื้ออะไรต่อ / งบเหลือเท่าไหร่ / อะไรต้องดู"). One primary action, a row of stat chips, optional context chip (e.g. selected slot) with a clear button. Zero-data state renders the page's entry point here (the same button the toolbar has — reuse the existing handler, e.g. `shopAddRowBtn.click()`).
2. **Control board** — the page's existing interactive control (Gear: the slot board) stays and keeps its behavior; visual polish only.
3. **Attention / Work Queue panel** — a compact card listing what needs the user next (queue of next actions + clickable issues). Every item must lead somewhere safe: jump to a row (`goToShopRow`-style), apply an existing filter (via a helper that **mirrors the existing chip/click handler**, never a parallel implementation), or select a context.
4. **Details section** — the dense table/list moves into a `<details class="shop-disclosure">`-style section with a labeled `<summary>` (+ a live count hint). Nothing inside is removed or re-rendered differently.

Hard rules (all load-bearing in 0.42 — carry them to every page):

- **No new math.** Hero/attention read values the page's existing update function already computed — stash them in one `pageStats`-style variable at the end of that function (Gear: `shopSummaryStats` in `updateShopSummary`) and render from it. Shared logic (e.g. the buy-next queue) is **extracted verbatim** into one helper both consumers call, never duplicated.
- **Render hooks ride the existing refresh path** — call the new `renderXHero()`/`renderXAttention()` at the end of the page's existing summary/update function so every existing mutation path refreshes them for free.
- **Simple/Advanced**: disclosure defaults follow `data-display` (Simple collapsed / Advanced open); the user's own toggle wins and persists in **one additive UI key per page** (`{ledgerOpen, tableOpen}`-shaped, tolerant load, `null` = follow mode; register in `APP_DATA_KEYS` + `UI_PREF_KEYS`). Controls that are power-user-only go in the `html[data-display="simple"]` CSS hide list (display-only — JS keeps rendering).
- **Deep links must open collapsed sections** — any jump-to-item path calls an `openXSection()` helper first (Gear: `openShopTableSection()` in `goToShopRow` + `focusShopRowName`).
- **`[hidden]` trap:** any new surface styled `display:flex/grid/inline-flex` needs an explicit `.class[hidden] { display: none; }` override (author display rules beat the UA `[hidden]` rule — bit us in 0.42 and earlier in the kwc compare bar).
- **Theming:** base styles in a new inline-CSS banner (RawBlock look), Modern overrides in a numbered section of `css/modern-theme.css`, and every new prominent class registered in **both** the Trust groups (index.html) and the Modern groups.
- **User text via `textContent` only**; escape/DOM-build everything in the new panels.
- Verify like 0.42: seeded-data Playwright pass asserting hero numbers equal the ledger/source-tab numbers computed by hand, all attention clicks, both display modes, all 3 themes, 480px, zero console errors.
